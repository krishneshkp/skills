# Platform Module: Webflow

Loaded when Webflow is detected (`data-wf-page` / `data-wf-domain` on `<html>`, assets from `website-files.com`, `w-` classes). Apply ON TOP of the core review.

Webflow's failure mode: **what the platform did that you need to catch** — defaults left on, toggles forgotten, automation quirks.

## Gate check

- **[CRITICAL] (production)** Production domain added to the site — this gates every indexability check. If only `.webflow.io` resolves, the site is not launched, whatever anyone says. [S]

## Settings & defaults — flag on sight

- **[HIGH]** "Asynchronously Load JavaScript" turned ON (Site settings → Publishing → Advanced publishing options; requires a paid Site plan — skip on free-plan staging reviews). [J] — check whether Webflow-generated script tags carry async/defer in the published HTML; if inconclusive, degrade to [MANUAL] rather than guessing.
- **[HIGH]** Minify CSS and JS toggles ON (published assets are `.min.` / minified; separate HTML/CSS/JS toggles under Site settings → Publishing → Advanced publishing options). [S]
- **[HIGH]** Global canonical tag has **no trailing slash** — Webflow appends each page's slug to it, so a trailing slash publishes double-slash canonicals (`site.com//contact`) on every subpage. Detectable: canonical URLs containing `//` after the protocol. [S]
- **[HIGH]** Webflow badge removed. [S]
- **[HIGH]** Default og:image / default favicon / default webclip not shipped. [S+J]
- **[LOW]** Default form names ("Email Form") replaced — they leak into submissions and integrations. [S]
- **[HIGH]** Google Fonts downloaded and uploaded to Webflow (or self-hosted) — never linked via Webflow's Google Fonts integration. Check for `fonts.googleapis.com` requests. [S]

## CMS & sitemap

- **[HIGH]** Reference/filter collection template pages: **publishing disabled** in the template page settings, so they never reach the sitemap or index. This is the Webflow-specific fix for the core sitemap check. [J]
- **[MANUAL]** CMS collections clearly named; fields have help text; required fields marked to prevent broken pages.

## Custom code & components

- **[HIGH]** Custom code placement correct: render-critical code in `<head>`, everything else before `</body>`; per-page code on pages, not global settings. [J] <!-- TODO(krishnesh): verify against a real launch -->
- **[HIGH]** Swiper instances: `a11y: true` and **unique data-attribute selectors** — never class-name selectors. [J]
- **[HIGH]** Background images have responsive variants — Webflow only generates srcset for inline images; large background images ship full-size to mobile. [S+J] <!-- TODO(krishnesh): verify against a real launch -->

## Platform behaviors to know (verified on real published output)

- Unset CMS link fields publish as hidden `href="#"` anchors with `w-condition-invisible` — expected output, not broken links (crawl.js excludes them).
- Staging (`.webflow.io`) noindex is served via robots.txt / X-Robots-Tag, **not** a meta tag — the absence of `<meta name="robots">` on staging pages is normal.
- Icon links are emitted `href`-before-`rel`; sitemap.xml often 404s on the staging subdomain even when auto-generation is on for the production domain.

## What the developer can and cannot change — never flag the "cannot" column

Before flagging any Webflow output, ask: **is there a Designer or site-setting that changes this?** If no, it is not a finding — platform-generated output the developer cannot edit or override gets, at most, a one-line informational note. Never assign it a severity or a fix.

Platform-generated, NOT findings:
- **Native Localization output** (verified against Webflow Help Center + community): hreflang tags are generated automatically per configured locale, including `x-default`, and are also injected into the auto-generated sitemap. Individual tags **cannot be edited or removed** from the Designer; the only controls are Settings → Localization: an auto-generation on/off toggle and the base-URL choice. Duplicated or repeated hreflang entries per locale are platform output, not a developer error — never a finding.
- **Forms publish with `method="get"`** (or no method): per Webflow's docs, new forms send submissions to Webflow by default and the bundled JS submits them without a page reload — the markup's method attribute alone is never a finding. The real check is a live submission with a visible success state.
- **Platform-bundled jQuery**: Webflow injects its own jQuery (v3.5.1 since 2020) into every published site; the developer cannot remove or upgrade it — the version is never a finding. (Loading a second jQuery via custom code IS a finding — the duplicate check in performance.md.) Same for `data-wf-*` attributes, `w-*` framework classes, and the generated `w-` form wrapper markup.
- **Interaction (IX2) inline styles** like `style="opacity:0"` in the initial HTML — generated by the animations engine.

Developer CAN change (fair game for findings): per-page titles/meta/OG, canonical base URL, minify + async-JS toggles, custom code (site-wide and per-page, head and body), alt text, form names and notification settings, collection-template publishing, custom font uploads, 301 redirects, robots.txt content, badge removal.
